# Mem0 — Technical Retrieval Analysis

- **Stars**: ~48K | **Funding**: $24M
- **Paper**: arXiv 2504.19413

## Retrieval Mechanism: Dual-path vector + parallel graph enrichment

### Primary: Vector similarity
- Each extracted memory fact embedded and stored in vector DB (Qdrant, Chroma, pgvector)
- Query embedded, cosine similarity finds top-k matches

### Secondary: Graph enrichment (Mem0g)
- Neo4j/Memgraph stores entity-relationship triples
- On retrieval: extract entities from query → find similar nodes via embedding → rerank with BM25 over triplet text
- Graph and vector run in parallel (ThreadPoolExecutor)
- Graph results APPENDED as `relations` array — do NOT reorder vector results

### What it does NOT do
- No graph traversal beyond 1-hop entity lookup
- No BFS/DFS
- No temporal windowing
- No spreading activation
- No fact consolidation/grouping
- Graph is side-channel enrichment, not retrieval driver
