## Режимы работы

### Ingest modes

| Флаг | Что происходит |
|------|---------------|
| `--ingest-mode dated` | `kb.add()` — сырой текст (sliding window chunks), **без LLM** |
| `--ingest-mode dated-learn` | `kb.add()` + `kb.learn()` — LLM fact extraction |

### Recall modes

| Переменная | Эффект |
|------------|--------|
| `AI_KNOT_PROVIDER=openai` | Авто-включает `llm_recall` (query expansion через LLM) |
| `AI_KNOT_LLM_RECALL=false` | Явно отключает `llm_recall`, перекрывает авто-включение |
| `AI_KNOT_LLM_RECALL=true` | Явно включает независимо от провайдера |

### Raw режим

```
--ingest-mode dated + AI_KNOT_LLM_RECALL=false
```
= ноль LLM вызовов, чистый BM25 + embeddings поиск.

### Независимость настроек

- `AI_KNOT_PROVIDER` влияет только на `llm_recall`, **не на `learn()` при ингесте**
- `--ingest-mode` контролирует только ингест, не recall
- Логика в `knowledge.py:109`:
  ```python
  self._llm_recall = llm_recall if llm_recall is not None else (provider is not None)
  ```

## Plan: фазы улучшений

### Phase 0 — MMR fix (реализована)

`src/ai_knot/knowledge.py:773` — заменить `max(jaccard, containment_a, containment_b)` на Jaccard only:

```python
union = len(ft | st)
sim = inter / union if union else 0.0
```

Эффект: learn-ON восстанавливается до v094 (+12-15pp).

### Phase 1 — Fuzzy dedup при add()

Near-duplicate detection: Jaccard нового контента с последними 5 фактами > 0.7 → skip.
Параметры:
- `_DEDUP_THRESHOLD = 0.7` — граница между overlap sliding window (0.72) и re-mention (0.3-0.5)
- `_DEDUP_WINDOW = 5` — окно stride-1: i±3 уже нет shared turns

Эффект: ~1700 → ~800 фактов, IDF удваивается для редких терминов. Cat1 +3-5pp.

### Phase 2 — Staged pipeline + Query Specificity

Заменяет hardcoded intent classifier на corpus-derived IDF specificity.

**Query Specificity:** `n_rare_tokens / len(tokens)` где rare = IDF > median_idf.

| Тип запроса | Specificity | Policy |
|-------------|-------------|--------|
| "What activities does X do?" | 0.25 | COVERAGE (80%) |
| "When did X sign up for pottery?" | 0.60 | RELEVANCE (60%) |
| "Would X enjoy Vivaldi?" | 0.50 | BALANCED |

**Stage 3 — IDF-weighted coverage greedy:**
```python
coverage_weight = max(0.1, 1.0 - specificity)
relevance_weight = max(0.1, specificity)
# rare tokens автоматически дают больше coverage credit через IDF
new_value = new_idf / total_idf
composite = pool_scores[fid] + coverage_weight * new_value
```

**Ожидаемый итог:**

## Tokenizer особенности

Porter stemmer **не идемпотентен**:
- "services" → "service" (strip -s) → "servic" (strip -e) — двойное применение меняет результат
- Queries и docs токенизируются по одному разу — это корректно, но важно не применять дважды
